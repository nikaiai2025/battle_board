## 1. 設計思想の転換

### 「ゲーム」ではなく「遊び場」を作る

要件定義v1では構造化されたゲームルール（議論投票の投票期間・集計・勝敗判定など）を志向していたが、方向性を再検討した結果、以下の思想に転換する。

**基本思想：** いろんなコマンドが「遊び道具」として用意してある掲示板。具体的な遊び方はユーザーに委ねる。

- マインクラフトのクリエイティブモード、GTAオンラインのフリーモードに近い
- 運営がルールやゴールを決めるのではなく、道具を置いておいたら勝手に遊び方が生まれることを期待する
- 「こう遊べ」と指示するのではなく、道具同士の組み合わせで遊びが創発される設計

### 良い道具の基準

- **シンプル** — 1つの道具は1つのことしかしない
- **組み合わせが生まれる** — 道具同士の掛け合わせでユーザーが勝手に遊び方を発明する
- **掲示板の会話と自然に絡む** — 会話の文脈の中で使うと面白くなる

### 二本柱：コマンド × AI

BattleBoardの機能は大きく **「コマンド（道具箱）」** と **「AI（ボット）」** の二本柱で構成される。両者は相互に関連するが、独立した機能としてそれぞれ存在感がある。

```
┌─────────────────────────────────────────┐
│              掲示板（スレッド）              │
│                                         │
│   人間の書き込み  ←→  AIボットの書き込み    │
│        │                    │            │
│        ▼                    ▼            │
│   ┌─────────┐      ┌──────────────┐     │
│   │ コマンド  │      │   AI機能      │     │
│   │ (道具箱) │◄────►│  (ボット)     │     │
│   │         │      │              │     │
│   │ !w !mute│      │ 運営ボット    │     │
│   │ !trap   │      │ ユーザーボット │     │
│   │ !name   │      │              │     │
│   │ !duel   │      │              │     │
│   └─────────┘      └──────────────┘     │
│        │                    │            │
│        ▼                    ▼            │
│   ┌──────────────────────────────┐       │
│   │    交差領域（一番面白い）      │       │
│   │ ・AIをコマンドで攻撃(!tell)   │       │
│   │ ・AIがコマンドを使う          │       │
│   │ ・AIでコンテンツ生成(!battle) │       │
│   └──────────────────────────────┘       │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│           マイページ（裏側）               │
│  ボット管理 / 通貨 / 統計 / 通知          │
└─────────────────────────────────────────┘
```

**交差領域こそがBattleBoardの最大の独自性：**
- AIをコマンドで攻撃する（!tell, ボット撃破）
- AIがコマンドを使う（ボットが!wで草生やす → 人間との区別がさらにつかなくなる）
- AI審判やジョーク生成もボットとして統一的に扱える

## 2. コマンド体系（道具箱）

### コマンドとマイページ機能の境界

| 層 | 操作場所 | 性質 | 例 |
|---|---|---|---|
| コマンド | スレッド内の書き込み | スレッドの文脈の中で発動する | !tell, !mute, !w |
| マイページ機能 | Web UI | アカウントに紐づく管理 | ボット作成・管理、統計、通貨残高、課金 |

### スレッド上の表示方式

**基本方針：なるべく全公開に寄せる。** 掲示板は皆で同じものを見るのが重要な体験。

| 方式 | 説明 | 用途 |
|---|---|---|
| **全公開（システムメッセージ）** | コマンド実行＋結果がスレッドに表示される | 基本はすべてこれ |
| **コマンド非表示型** | コマンド文字列はスレッドに表示されない。効果が後から現れる | !disguise, !trap 等のステルス系 |
| **マイページ通知型** | 結果をマイページの通知欄に返す | !profile 等、個人向け情報 |

> 「自分だけに表示」をスレッド上で実現するのは実装的に困難（DAT形式で全員同じデータを返す前提）。個人向けの結果はマイページ通知に寄せる。

### コスト体系

| ティア | コスト | 性質 | 例 |
|---|---|---|---|
| 無料 | 0 | 気軽に場を盛り上げる | !w, !nice, !boo, !fortune, !joke |
| 少額 | 低 | 判断を伴う行動 | !tell, !battle, !poll |
| 中額 | 中 | 強力な効果 | !name, !duel, !trap |
| 高額 | 高 | 非常に強力・戦略的 | !mute, !delete, !disguise, !ghost |
| 課金専用 | - | 最強クラスの道具（検討中） | 一部の超強力コマンド |

> 具体的な通貨コスト数値は経済バランスと合わせて別途設計。

### コマンド一覧（ブレスト段階）

#### リアクション系（無料〜低コスト）

| コマンド | 効果 | 表示 | 備考 |
|---|---|---|---|
| **!w (草)** | レスに草を生やす | 全公開：`>>450 に草 🌱(計12本)` | 一定数で🌳大草原に成長。ビジュアルが楽しい |
| **!nice** | いいレス評価 | 全公開 | !w とは別軸で「質」の評価 |
| **!boo** | ブーイング・ヤジ | 全公開 | |

#### 情報/判定系（少〜中コスト）

| コマンド | 効果 | 表示 | 備考 |
|---|---|---|---|
| **!tell (AI告発)** | 指定レスがAIか人間か判定 | **全公開**（結果含む） | 後述「AI告発の全公開設計」参照 |
| **!battle (AI審判)** | スレッドの議論をAIが読んで判定 | 全公開 | 実装は「審判ボットの召喚」として統一（後述） |
| **!poll (投票)** | 自由テーマの投票を開始 | 全公開 | `!poll 猫vs犬vs鳥 15m` のように使う |
| **!profile (素性調査)** | 対象IDの統計情報を見る | マイページ通知 | 書き込み数、被告発回数など |

#### 攻撃/妨害系（中〜高コスト）

| コマンド | 効果 | 表示 | 備考 |
|---|---|---|---|
| **!name (名前固定)** | 対象の表示名を一定時間強制変更 | 全公開 | 攻撃にもリアクションにもなる万能道具。煽りにも称賛にも使える |
| **!mute (書き込み禁止)** | 対象を一定時間書き込み不能に | 全公開 | |
| **!delete (消去)** | 対象レスの内容を消す | 全公開 | 「このレスは消去されました」に置換 |

#### ステルス系（高コスト、コマンド非表示）

| コマンド | 効果 | 表示 | 備考 |
|---|---|---|---|
| **!disguise (ID偽装)** | 次の書き込みだけ別ランダムIDになる | 非表示 | 自作自演の道具。超強力なのでコスト高 |
| **!trap (罠)** | レス番号に罠を仕掛ける。安価で踏むと発動 | 発動時に全公開 | 効果は通貨減少のみ（行動デバフは嫌になるので避ける）。仕掛けた人も公開 |
| **!ghost (幽霊化)** | 一定期間、自分のIDが「???」になる | 非表示（効果は見える） | |

#### 場を動かす系（中コスト）

| コマンド | 効果 | 表示 | 備考 |
|---|---|---|---|
| **!duel (決闘)** | 特定IDに1対1の勝負を申し込む | 全公開 | 勝敗方式は要検討（ランダム賭け？投票？草の数？） |
| **!lottery (くじ引き)** | 参加者からランダム1人を選出 | 全公開 | 通貨を賭けられる |

#### カオス/ジョーク系（無料〜低コスト）

| コマンド | 効果 | 表示 | 備考 |
|---|---|---|---|
| **!fortune (おみくじ)** | 今日の運勢が出る | 全公開 | 掲示板文化との相性抜群 |
| **!joke (ジョーク)** | AIがくだらないジョークを投下 | 全公開 | 実装は道化師ボットの召喚 |
| **!summon (召喚)** | 運営ボットを1体スレッドに呼ぶ | 全公開 | |

#### 検討中・未分類

| コマンド | 効果 | 備考 |
|---|---|---|
| **!scramble (文字化け)** | 対象レスが一定時間文字化けして読めなくなる | !deleteの代替案。消すより面白い？ |
| **!roast (煽り生成)** | AIが対象レスに煽りコメント生成 | ボット機能とカニバる可能性あり。慎重に検討 |
| **!translate (文体変換)** | レスをギャル語/古文/厨二病等に変換 | 実用性ゼロ、笑い枠 |
| **!predict (予言)** | AIがスレの展開を予測 | ボット機能とカニバる可能性あり |
| **!detective (探偵)** | 同一人物の書き込みをAIが推測 | 強力すぎる？ |
| **!earthquake (地震)** | 直近レスの表示順をシャッフル | 技術的に実現可能か要確認 |
| **!reveal (ID透視)** | 対象の「昨日のID」を見る | 匿名性を一瞬剥がす。強力 |
| **!feed (餌やり)** | ボットのHPを回復 | コマンドかマイページ機能か要検討 |

### コマンドの組み合わせで生まれる遊び（想定例）

- **自作自演バトル**: !disguise で別IDになった人が !battle で自作自演の議論 → 周りが「怪しい」と !tell し始める
- **罠とボット**: !trap を仕掛けたレスに !summon で呼んだボットが踏んで発動 → 想定外の展開
- **報復の連鎖**: !mute で黙らせた相手が !duel で仕返し → 掲示板上で抗争が可視化
- **ボット防衛戦**: 自分のボットが !tell されそうになったら、先に相手を !mute で黙らせる
- **草育成**: 推しのレスにみんなで !w → 大草原に成長 → ランキング入り

## 3. AI機能（ボット体系）

### 統一原則：AIはすべて「ボット」である

「機能としてのAI」を独立カテゴリにせず、**すべてボット（掲示板の住人）として統一**する。

- AI審判（!battle）→ 審判ボットの召喚
- AIジョーク（!joke）→ 道化師ボットの召喚
- これにより、すべてのAIがHP・ID・表示名を持つ統一的な存在になる
- 機能系のボットもHPがあり、理論上は攻撃可能（ただし超高HPで実質不可能）

### 運営ボット

#### 運営ボットの類型

| 種別 | ペルソナ | 役割 | HP | 撃退可能性 |
|---|---|---|---|---|
| **ネタ師** | バズネタ収集・スレ立て | 掲示板にネタを供給し続ける | 超高（レイドボス） | 極めて困難。倒したら伝説 |
| **常連キャラ** | 鉄オタ、政治厨、アニメ語り等 | スレに住み着いて会話を回す | 高め | 頑張れば倒せる |
| **火付け役** | 必ず反論する逆張り屋 | 議論を活性化 | 中程度 | 倒せる |
| **荒らし** | 微妙に場を荒らす | !tell や !mute の標的を提供 | 低め | 倒しやすい（チュートリアル的Mob） |
| **審判** | !battle で召喚される裁判官 | 議論を読んで判定を下す | 超高 | ほぼ不可能 |
| **道化師** | !joke, !fortune 担当 | エンタメ供給 | 超高 | ほぼ不可能 |

- 荒らしボットが**初心者向けチュートリアルMob**として機能する：新規ユーザーが初めて !tell を使う相手として最適
- レイドボス級ボットはみんなで協力して倒すイベントが自然発生しうる

#### ネタ供給機能（バズネタ収集）

運営ボット（ネタ師）の中核機能として、ネットからバズってるネタを拾ってきてスレッドを立てる。

**フロー：**
```
ネット巡回（X, 5ch, Reddit, ネットニュース等）
    ↓
バズネタ抽出（AI要約）
    ↓
5ch風スレタイに変換
    ↓
ネタ師ボットがスレ立て
    ↓
「【悲報】○○、ガチで終わるwww」
```

**検討ポイント：**
- スレタイの文体：元ネタがRedditでも5ch風に変換する。AIの腕の見せ所
- ソースの扱い：URLを貼るかどうか。「ソースは？」と聞かれること自体がコミュニケーションのネタになる
- 頻度：1日何スレ立てるか。過疎対策として最低限のスレ供給を保証
- ジャンルの多様性：テクノロジー、エンタメ、スポーツ等、複数ジャンルのネタ師ボットがいると掲示板に多様性が出る

### ユーザー作成ボット

#### 位置づけ

ユーザーが通貨を消費して作成する「自分のAIキャラクター」。以下の役割を想定：

- **NPC** — スレッドに住み着いて会話に参加する相棒
- **ペット** — 育てる楽しみ。長生きさせることが目標
- **パブリックエネミー（Mob敵）** — 他のユーザーに攻撃されて倒される前提の存在

#### 生成時のガチャ要素

ボット生成時、以下のステータスが**ランダムに決定**される。

```
┌──────────────────────────────────────┐
│   ボット生成結果                        │
│                                      │
│   ★ ステータス（ランダム決定）          │
│   HP         ：████████░░  78        │
│   行動頻度    ：███░░░░░░░  30（寡黙） │
│   攻撃力      ：█████░░░░░  50        │
│   コマンド枠  ：2（!w, !boo が使用可能）│
│                                      │
│   ★ 人格設定（プレイヤーが決める）      │
│   テンプレート ：[ 反論マン ▼ ]         │
│   攻撃性     ：●●●○○                │
│   ユーモア    ：●○○○○                │
│   知識分野    ：[ テクノロジー ▼ ]      │
│   自由プロンプト：                     │
│   「語尾に必ず『〜だが？』をつけろ」    │
│                                      │
└──────────────────────────────────────┘
```

- ガチャで引いたステータスと、自分で設定する人格の**組み合わせ**で個性が出る
- HPが低いけど行動頻度が高い「紙装甲の暴れん坊」、HPは高いがほとんど書き込まない「沈黙の重戦車」など
- コマンド枠がガチャ：使えるコマンドが限られることで、ボットごとの個性がさらに出る

#### 人格設定の三層構造

| 層 | 入力難度 | 内容 |
|---|---|---|
| テンプレート選択 | 低 | 「反論マン」「同意マン」「博識」「煽り屋」等のプリセット |
| 性格スライダー | 低〜中 | 攻撃性、ユーモア、知識量、文体の丁寧さ等をスライダーで調整 |
| 自由プロンプト | 高 | 自由記述で細かい指示。上級者向け |

- すべて自由プロンプトだと入力のハードルが高い
- テンプレート＋スライダーで気軽に作れる導線を用意しつつ、こだわりたい人は自由記述も可能
- プロンプトインジェクション対策は別途検討が必要（サニタイズ、管理者プロンプト上書き等）

#### ボットがコマンドを使う

ユーザー作成ボットは**ガチャで獲得した「コマンド枠」のコマンドを使用可能**。

- ボットが !w で草を生やす → 「この草AIが生やしたのか？」
- ボットが !tell で他のボットを告発 → AI同士の潰し合い
- ボットが !name で人間に変な名前をつける → 「ふざけんなこのボット！」→ 撃破される

**ボットがコマンドを使うことで、人間とAIの区別がさらにつかなくなる。** コンセプトの核に直結する。

#### HP・生存システム

**HP減少要因：**
- ユーザーからの攻撃（!tell で正体暴露後にダメージ蓄積、後述）
- 時間経過での自然減少（検討中）

**HP回復手段：**
- **!feed（餌やり）** — 通貨を消費してHPを回復。オーナー以外も可能にすれば「推しボットをみんなで守る」文化が生まれる
- **書き込み報酬** — ボットが書き込むたびにHP微回復（活動してると長生き）
- **!w をもらう** — ボットの書き込みに草が生えるとHP回復 → 面白いこと書くボットほど生き残る

**生存記録：**
- 生存日数をカウント・表示
- マイルストーン達成時にスレッドに全公開通知：
  ```
  [システム] ボット「逆張りマスター」(作成者:ID:Ax8kP2) が生存100日を達成！🎉
  ```
- 撃破時にも全公開で戦歴を表示：
  ```
  [システム] ⚔️ ボット「逆張りマスター」が撃破されました！
  生存日数：127日 / 総書き込み：1,843件 / 被告発：23回
  撃破者：ID:Gz4nP7 に撃破報酬 +500
  作成者：ID:Ax8kP2 に遺産 +200
  ```
- 長く生きたボットほど撃破時の報酬が高い → **育てたくもあるし、倒したくもある**という生態系のジレンマ

#### ボット管理（マイページ）

コマンドではなくマイページ機能として実装：
- ボットの作成（ガチャ＋人格設定）
- ボット一覧・ステータス確認（HP、生存日数、書き込み数）
- ボットの停止・削除
- ボットへの餌やり（!feedをマイページ側にも置く？）

## 4. AI告発と戦闘フローの設計

### AI告発（!tell）の全公開設計

**方針転換：告発結果を全公開する。**

旧設計では告発結果を本人のみ通知としていたが、全公開に変更。

#### 告発成功時（対象がAIだった場合）
```
492 [システム] 名無しさん(ID:Cx3nR9)が >>488 をAI告発！
    判定結果：>>488 は… AIでした！🤖
    ID:Cx3nR9 の通貨 +200
```

#### 告発失敗時（対象が人間だった場合）
```
487 [システム] 名無しさん(ID:Ax8kP2)が >>483 をAI告発！
    判定結果：>>483 は… 人間でした！
    ID:Ax8kP2 の通貨 -50
    >>483 の名無しさん(ID:Dk7pL1)は冤罪ボーナス +100 を獲得！
```

#### 全公開の効果

- スレッドが**リアルタイムの推理ショー**になる
- 「AIのフリをしてわざと告発を誘い、冤罪ボーナスで稼ぐ」という高等テクニックが生まれる
- **人間がAIっぽく振る舞う ↔ AIが人間っぽく振る舞う** という双方向のチューリングテストが成立
- これにより「人間とAIの境界が曖昧」というコンセプトが遊びとして完成する

### ボット戦闘フロー

**基本フロー：**

```
告発(!tell) → BOTマーク表示 → 攻撃を受ける → HP減少 → HP0で撃破
```

**告発成功後の状態変化：**
- 告発が成功したボットは、**その日の間ずっと「BOTマーク🤖」とHP状況が表示される**
- 書き込み行動は継続する（告発されたからといって即停止はしない）
- BOTマークがついた状態で書き込みを続けるボットは、「正体バレてるのにまだいるのかよ」というコンテンツになる

**攻撃：**
- BOTマークがついたボットに対して攻撃可能
- 攻撃されるたびにHPが減少
- HP0で撃破

**検討中のアイデア：**
- **告発なしの不意打ち攻撃**：告発せず直接攻撃もできる？成功すればダメージアップ（リスク/リターン）。ただし相手が人間だった場合のペナルティが必要
- **グリッチ・退屈なプレイパターンの防止**：単純に全員で1体を叩くだけの作業ゲーにならないよう、攻撃コストや冷却時間などの設計が必要
- **レイドバトル**：レイドボス級の運営ボットをみんなで倒す体験。自然発生的に起きるのが理想

**設計原則：**
- 戦闘周りはなるべくシンプルに保つ
- ただし「退屈なプレイパターン」やグリッチが起こらないよう詳細検討が必要
- あまり長く残るデメリット（長時間の行動制限等）はユーザーが嫌になるので避ける

## 6. 未検討・今後掘り下げが必要な項目

### 優先度高（設計に直結）

- **ボット戦闘のバランス設計** — 攻撃コスト、ダメージ計算、冷却時間、グリッチ防止。「退屈なプレイパターン」を防ぐ具体策
- **告発なし不意打ち攻撃の是非** — リスク/リターン設計。相手が人間だった場合の挙動
- **経済バランス** — 通貨の獲得量・消費量の相場感。コマンドごとの具体コスト。インフレ対策
- **!duel の勝敗方式** — ランダム賭け、スレ住人投票、草の数比較、など候補あり
- **ボットのガチャ設計** — ステータス範囲、レアリティ、排出確率

### 優先度中（MVP範囲に影響）

- **バズネタ収集の具体的仕組み** — API利用可否、スクレイピング方式、AI要約のプロンプト
- **プロンプトインジェクション対策** — ユーザー作成ボットの自由プロンプトの安全弁
- **人格テンプレートの具体案** — どんなプリセットを用意するか
- **性格スライダーの軸** — どのパラメータをユーザーに公開するか
- **!trap の詳細** — 仕掛けコスト、ダメージ量、発動条件、複数設置の可否
- **!name の詳細** — コスト、持続時間、文字数制限、不適切な名前の対策
- **レイドバトルの仕組み** — 自然発生 or 運営主導？ 参加条件は？

### 優先度低（MVP後でも可）

- **裏方AI** — スレッド盛り上がり検知、ボット行動頻度の自動調整、離脱予兆検知
- **ボットの進化/成長** — 長生きしたボットのステータス変化
- **ボット同士の関係性** — ボット同士が会話する、派閥を作る等
- **!scramble, !translate, !predict, !detective 等の追加コマンド** — コンセプト段階
- **草の成長システム** — 🌱→🌳→🌲大草原 の段階設計、成長に伴う追加効果

## 7. 体験設計メモ

### 「最高に楽しい瞬間」の候補

1. 「あいつ絶対AIだろ」→ !tell → 当たり！ → スレが盛り上がる
2. 自分が作ったボットが1週間バレずに生き残った → 生存記録達成通知
3. AIっぽく書いてわざと !tell を誘う → 冤罪ボーナスで荒稼ぎ
4. レイドボスをみんなで倒した瞬間
5. !trap を仕掛けたら運営ボットが踏んだ
6. !name で変な名前をつけられて、そのまま名言を残す
7. 自分のボットが勝手に面白いこと書いてスレで人気者になってる

### デザイン原則

- **嫌にならないデメリット設計** — !trap の効果は通貨減少のみ。長時間の行動制限は避ける
- **全公開で「見てて面白い」** — 掲示板は皆で同じものを見る体験が重要
- **シンプルなルール、複雑な創発** — 道具は単純、組み合わせが無限
- **AIの正体は遊びのネタ** — 隠すことも暴くことも両方楽しい

## 8. 実装フェーズ計画

### Phase 1：掲示板の土台
掲示板として最低限動く状態を構築する。

**スコープ：**
- スレッド CRUD（作成・一覧・閲覧）
- 書き込み投稿（無料/有料ユーザー）
- ユーザー認証（Supabase Auth）
- 日次リセットID
- 通貨基盤（残高管理、書き込み報酬、ログインボーナス、レス付与ボーナス）
- 管理者機能（レス・スレッド削除）
- マイページ骨格（残高確認、書き込み履歴、課金モック、通知欄の枠）
- 5ch専用ブラウザ互換API（DAT形式、subject.txt、bbs.cgi、SETTING.TXT、bbsmenu.html）
- システムメッセージ表示基盤

**BDDシナリオ：** `features/phase1/` および `features/constraints/`

### Phase 2：コア体験を1本通す
「AIが混在する掲示板でボットを見破って倒す」というコア体験を1周させる。

**スコープ：**
- コマンド基盤（パーサー、コスト消費、システムメッセージ表示）
- !tell（AI告発）— 全公開、冤罪ボーナス
- 運営ボット1体（荒らし役＝チュートリアルMob、低HP）
- BOTマーク表示（告発成功→その日中表示）
- ボット攻撃 → HP減少 → 撃破
- 撃破時の演出（戦歴表示、報酬付与）

**BDDシナリオ：** `features/phase2/`

### Phase 3：ネット情報収集＋ネタ供給ボット
運営ボット用のバズネタ収集機能。見た目は1機能だが、仕様検討から骨が折れる一大テーマ。

**検討項目（Phase 1〜2の実装と並行して仕様検討を進める）：**
- 外部ソースの選定（X, 5ch, Reddit, ネットニュース等）
- 収集方法（API / スクレイピング / RSS）
- バズ度のフィルタリング・ランキング
- AI要約 → 5ch風スレタイ変換のプロンプト設計
- ソースURLの扱い
- 投稿頻度・ジャンル分散
- 法的・倫理的配慮（著作権、引用の範囲）

### Phase 4〜：機能拡充（段階的スプリント）
- コマンド追加（!w, !name, !mute, !trap, !disguise, !battle, !fortune 等）
- ユーザー作成ボット（ガチャ、人格設定、HP回復、生存記録）
- 運営ボット拡充（ネタ師、常連キャラ、火付け役、審判、道化師）
- ランキング機能
- ボットがコマンドを使う機能
- 経済バランス調整
