


ChMateやSikiといった既存の5ch専用ブラウザ（専ブラ）からアクセス可能な「5ch（旧2ch）互換掲示板」を自作するための仕様レポートをまとめました。

これらの専ブラは、現在でも**「伝統的な2ch互換API（ファイル群とCGI）」**をサポートしており、仕様に沿った構成をサーバー上に構築すれば、URLを指定するだけでネイティブアプリから読み書きが可能になります。

以下に、準拠すべき必須条件と詳細な仕様を解説します。

---

### 1. ディレクトリ構造と必須エンドポイント
専ブラは、登録された「板のURL」を基準にして各ファイルのパスを自動的に推論し、GET/POSTリクエストを送信します。
ドメインが `https://example.com`、板のID（ディレクトリ名）が `testboard` の場合、以下のディレクトリ構造とファイルが必要です。

```text
example.com/
 ├ testboard/              ← 板のルートURL（専ブラに登録するURL）
 │  ├ subject.txt        (GET) スレッド一覧
 │  ├ SETTING.TXT        (GET) 掲示板の設定情報
 │  └ dat/
 │     ├ 1234567890.dat  (GET) スレッドごとの書き込みデータ
 │     └ 1234567891.dat
 └ test/
    └ bbs.cgi            (POST) 新規スレッド作成・レス書き込み用エンドポイント
```
※ `bbs.cgi` は、歴史的な経緯から板ディレクトリと同階層の `test/` ディレクトリ直下に置くのが最も確実です（多くの専ブラがこのパスをハードコード、もしくは標準として推測します）。

---

### 2. 各ファイルのフォーマット仕様

専ブラからのアクセス（GETリクエスト）に対して返す必要のある静的ファイル（あるいは動的生成結果）の仕様です。

#### ① 文字コード（超重要）
ChMateをはじめとする大半の専ブラは、レスポンスの文字コードが **Shift-JIS (CP932 / Windows-31J)** であることを前提として動きます。SikiなどのモダンなアプリはUTF-8に対応している場合もありますが、汎用性を考慮するならファイル出力・レスポンスはすべて `Shift_JIS` に統一することを強く推奨します。

#### ② `SETTING.TXT`（掲示板設定）
板の名前や挙動を専ブラに伝えるためのテキストファイルです。改行区切りで `キー=値` の形式で記述します。
最低限、以下の記述があれば動作します。
```text
BBS_TITLE=自作テスト掲示板
BBS_SUBJECT_COLOR=#FF0000
BBS_TEXT_COLOR=#000000
```

#### ③ `subject.txt`（スレッド一覧）
板に存在するスレッドの一覧を提供するファイルです。
*   **フォーマット**: `[スレッドID].dat<>[スレッドタイトル] ([レス数])\n`
*   **スレッドID**: 基本的にスレッドが作成された時の「UNIXタイムスタンプ（10桁）」を使用します。
```text
1234567890.dat<>最初のテストスレッド (5)
1234567891.dat<>２番目のスレッドです (12)
```

#### ④ `.dat`ファイル（スレッド本文データ）
スレッド内の全レスが記録されたファイルです。1行が1レスに対応します。
*   **フォーマット**: `名前<>メール<>日付とID<>本文<>スレッドタイトル\n`
*   **各要素のルール**:
    *   **名前**: デフォルト名などは `<b>`タグ等を含めることがありますが平文でも可。
    *   **メール**: `sage` などのコマンドやメールアドレス。
    *   **日付とID**: `2026/03/03(火) 12:00:00.00 ID:AbCdEfGh` のように、厳密に `YYYY/MM/DD(曜日) HH:MM:SS.ss ID:xxxx` のフォーマットに従う必要があります（専ブラの正規表現パーサーがここでコケると表示が崩れます）。
    *   **本文**: 改行は `<br>` に変換します。`<` や `>` などのHTML特殊文字は `&lt;` `&gt;` にエスケープ必須です。
    *   **スレッドタイトル**: **「1レス目（>>1）」の末尾にのみ**記述します。2レス目以降は空（`<>`で終わる）にします。

**【出力例: 1234567890.dat】**
```text
名無しさん<>sage<>2026/03/03(火) 12:00:00.00 ID:12345678<>テストの本文です。<br>よろしく。<>最初のテストスレッド
名無しさん<>sage<>2026/03/03(火) 12:05:00.00 ID:87654321<>２ゲットです。<>
```

### 3. 書き込みAPI（bbs.cgi）の仕様

専ブラの「書き込む」ボタンを押した際、アプリは `https://example.com/test/bbs.cgi` に対して `application/x-www-form-urlencoded` 形式で **POST** リクエストを送信します。

#### POSTパラメータ
| パラメータ名 | 説明 | 例 |
| :--- | :--- | :--- |
| `bbs` | 書き込み先の板ID | `testboard` |
| `key` | スレッドID。新規スレ立て時は空 | `1234567890` (レス時) |
| `time` | 専ブラが取得した現在時刻(UNIXタイム) | `1709439600` |
| `submit` | 送信ボタンの値（一部専ブラで必須） | `書き込む` (Shift-JISエンコード) |
| `FROM` | 名前欄の入力値 | `名無しさん` |
| `mail` | メール欄の入力値 | `sage` |
| `MESSAGE` | 本文。改行はそのまま `\n` または `\r\n` | `テスト書き込みです` |
| `subject` | スレタイ。新規スレ立て時のみ入力される | `最初のテストスレッド` |

#### レスポンス（書き込み結果）
書き込み処理が成功した場合、通常は `200 OK` とともに、適当な成功HTML（例: `<html><body>書き込みが完了しました。</body></html>`）を返します。
※エラーが生じた場合（文字数オーバー、連投規制など）は、レスポンスのHTMLの本文に「ERROR:」という文字列とエラー理由を含めておくと、専ブラ側がそれを検知してエラーポップアップとしてユーザーに表示してくれます。

### 4. 専ブラ（ChMate・Siki）側での登録・アクセス方法

構築したシステムを専ブラで読み込ませるアプローチは2種類あります。

**A. URLを直接指定して開く（板の単体登録）**
ユーザーに直接 `https://example.com/testboard/` をChMateの「URLを指定して開く」や、Sikiの「カスタム掲示板追加」に入力させる方法です。設定が正しければ即座にスレッド一覧がパースされます。

**B. `bbsmenu.html` を用意する（汎用メニュー登録）**
複数の板を運営する場合や、専ブラの「板一覧（BBSMENU）」に自然に組み込みたい場合は、サーバーの任意の場所に `bbsmenu.html` を作成します。
```html
<HTML>
<BODY>
<B>自作カテゴリ</B><br>
<A HREF="https://example.com/testboard/">テスト掲示板</A><br>
<A HREF="https://example.com/subboard/">サブ掲示板</A><br>
</BODY>
</HTML>
```
ユーザーがChMate等の設定画面から「BBSMENUのURL」にこの `bbsmenu.html` のURLを登録すると、自動的にすべての板がリストに表示されるようになります。

### まとめと実装上のアドバイス

1. **文字コード（Shift-JIS）への変換**
   DB（MySQL等）や内部ロジックはUTF-8で組み、出力時（`subject.txt`, `.dat` の生成）および 入力時（`bbs.cgi` でのPOST受け取り）の境界部分でのみ Shift-JIS にエンコード／デコードする設計にすると現代のフレームワーク（Node.js, Python, PHPなど）でも作りやすいです。
2. **ルーティングによる偽装**
   実際にサーバー上に `.txt` や `.dat` という物理ファイルを置く必要はありません。Webフレームワークのルーティングを利用して、`/testboard/dat/1234567890.dat` というGETリクエストに対してDBから該当スレッドを引き出し、動的にShift-JISの文字列を生成して `Content-Type: text/plain; charset=Shift_JIS` で返却すれば完全に動作します。
3. **User-Agentによるアクセス制御**
   専ブラからのアクセスには特有のUser-Agent（例: `Monazilla/1.00`, ChMate独自のUAなど）が付与されます。これを利用して、通常のブラウザアクセスとAPIアクセスの挙動を分けることも可能です。
