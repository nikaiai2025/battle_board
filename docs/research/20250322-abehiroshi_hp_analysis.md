# 阿部寛のホームページ HTML構造分析

> 分析対象: https://abehiroshi.la.coocan.jp/
> 分析日: 2025-03-22
> 目的: 「現代のブラウザで見ても当時っぽく見える」理由の特定と、開発連絡板UIへの応用

## 1. ファイル構成

| ファイル | 役割 | サイズ |
|---|---|---|
| `index.htm` | frameset定義のみ（実質5行） | 極小 |
| `menu.htm` | 左フレーム（ナビゲーション） | 約2KB |
| `top.htm` | 右フレーム（メインコンテンツ） | 約4KB |
| `abe-top-20190328-2.jpg` | プロフィール写真 350x414px | - |
| `abehiroshi.jpg` | 背景タイル画像 | - |

制作ツール: JustSystems Homepage Builder Version 20〜23

## 2. フレーム構造

```html
<frameset cols="18,82">
  <frame src="menu.htm" marginheight="0" marginwidth="0" scrolling="auto" name="left">
  <frame src="top.htm"  marginheight="0" marginwidth="0" scrolling="auto" name="right">
</frameset>
```

- **比率指定** `cols="18,82"`（ピクセル固定ではない）
- 1920px画面: 左 345px / 右 1575px
- 1024px画面: 左 184px / 右 840px
- **画面サイズに追従する**ため、どの解像度でもバランスが崩れない

## 3. 左メニュー (menu.htm)

```html
<body bgcolor="#F0F0FF" marginwidth="0" marginheight="0">
```

- 背景色: `#F0F0FF`（極薄ラベンダー）
- テーブル: `border="0" width="142"` の3カラム構成

### カラム構造

| 列 | 幅 | 内容 |
|---|---|---|
| 1列目 | 15px | 空白（インデント） |
| 2列目 | 10px | 色付き記号「■」（各項目で異なる色） |
| 3列目 | 148px | リンクテキスト |

### メニュー項目の記号色

| 項目 | 色コード | 色 |
|---|---|---|
| トップ | `#FFCCCC` | ピンク |
| 映画出演 | `#00FF00` | グリーン |
| ドラマ出演 | `#33FFFF` | シアン |
| 舞台出演 | `#0099FF` | ブルー |
| 写真集 | `#0000FF` | ダークブルー |
| 出版本 | `#333399` | ネイビー |
| 管理人 | `#CC00CC` | マゼンタ |

- リンクに `target="right"` を指定（右フレームに表示）
- 各項目間にスペーサー行（空セル）を挿入して行間を確保

## 4. メインコンテンツ (top.htm)

```html
<body background="abehiroshi.jpg" marginwidth="0" marginheight="0">
```

- 背景: タイル敷き画像
- ページタイトル: `<h1 align="center">`（ブラウザデフォルトの32px相当）
- コンテンツ配置: `<table align="center">`
- プロフィール画像: `width="350" height="414" border="0"`
- プロフィール情報テーブル: `width="256"`（14px空白列 + 230px本文列）
- 区切り線: `<hr width="100%" size="1">`
- 警告文: `<font color="#ff0000">` + `<b>`
- 連絡先に `mailto:` リンク
- 下部に `<br>` を大量に並べてフッター余白を確保（CSSのmargin-bottom代替）

## 5. 最重要発見: font-size指定が一切ない

**3ファイルすべてで、font-sizeの指定が一切存在しない。**

- CSSの `font-size` → 不使用
- `<font size="X">` → 不使用
- `<style>` タグ → 不使用（CSS自体がゼロ）

すべてのテキストがブラウザデフォルトで描画されている:

| 要素 | ブラウザデフォルト |
|---|---|
| body（本文） | **16px** |
| `<h1>` | **32px**（2em） |
| `<p>` | **16px** + 上下マージン |
| `<a>` | 16px + 青色下線 |

## 6. 使われているCSS

**ゼロ。** `<style>` タグも `style=` 属性もCSSファイルへの `<link>` も一切ない。
すべての装飾はHTML属性（`bgcolor`, `color`, `align`, `width`, `border`）で行われている。

## 7. なぜ現代のブラウザでも「当時っぽく見える」のか

### 7.1 フォントサイズがブラウザデフォルト (16px)

1990年代のブラウザも現代のブラウザも、デフォルトのフォントサイズは16pxで一貫している。
つまり「当時のブラウザで見たときのサイズ感」と「現代のブラウザで見たときのサイズ感」が完全に一致する。

一方、`font-size: 12px` と明示指定した場合:
- 当時の96dpi/800x600モニター → 画面の1.5%相当 → 読みやすい
- 現代の144dpi/1920x1080モニター → 画面の0.6%相当 → 米粒

**12pxの「レトロ感」は当時のモニター環境に依存しており、現代モニターでは再現されない。**

### 7.2 比率指定のフレーム

`frameset cols="18,82"` はパーセンテージ比率のため、画面幅が変わっても左右の比率が保たれる。
800px画面でも1920px画面でも「左18%にメニュー、右82%にコンテンツ」という見た目が維持される。

### 7.3 装飾の不在

CSSが一切ないため「現代的な装飾（角丸、影、グラデーション）が入り込む余地がない」。
逆に「レトロな装飾を頑張って入れる」と、その装飾自体が嘘くさくなるリスクがある。

### 7.4 テーブルレイアウトの素朴さ

テーブルの列幅が固定ピクセルで、CSSリセットやbox-sizingの影響を受けない。
20年以上前のHTMLがそのまま意図通りに描画される堅牢性がある。

## 8. 開発連絡板への応用提案

| 項目 | 現在の開発連絡板 | 阿部寛HPの知見 | 提案 |
|---|---|---|---|
| フォントサイズ | 12px固定 | **指定なし（=16px）** | 14〜16pxに変更。最大の効果 |
| フレーム比率 | 160px固定 / 640px固定 | 18:82の比率指定 | 比率指定への変更を検討 |
| 左ナビ背景色 | `#c0c8d8` | `#F0F0FF`（極薄ラベンダー） | より淡い色に変更を検討 |
| メニュー記号 | なし | 色付き「■」（項目ごとに異なる色） | 記号の追加を検討 |
| CSS量 | 多い | ゼロ | 過剰な装飾の削減を検討 |
