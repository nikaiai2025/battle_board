# おーぷん2ちゃんねる `なんでも実況(ジュピター)` UI分析

> 分析日: 2026-04-20
> 目的: 「見た目が成立している理由」を抽出し、BattleBoard の匿名掲示板 UI に転用できる形へ整理する

## 参考サイトの視覚的特徴

- 淡い黄緑のテーブル箱を `95%` 幅で中央に積む構成で、ページ全体に一貫した「掲示板の板感」がある。背景は細かなベージュ系タイルで、真っ白より生活感が出ている。
- 文字サイズは全体に小さいが、役割差が明確。通常情報は `9pt` から `10pt`、スレタイだけ `font size=5` の赤で大きくしており、どこを見るべきか迷いにくい。
- 色数は多くない。基調は灰・薄緑・薄黄で、赤はスレタイや加速度、黄は勝敗や注目状態など「今ここを見る」箇所に限定している。
- スレ一覧は「スレタイ」だけでなく、左端の小さな情報帯で `人数` と `勢い` を常時見せている。小さい面積で活気の差を比較できる。
- ヘッドラインは高さ固定のスクロール領域で、経過秒・増加レス数・タイトル・抜粋を圧縮表示している。新着の流れが独立した情報面になっている。
- 一覧と個別スレの両方で、箱の中に本文まで少し見せる構成になっている。リンク集だけで終わらず、会話の温度が最初から伝わる。
- 装飾は素朴だが、余白の切り方は整理されている。箱と箱の間隔、見出し直下の空き、操作リンクの低い主張が一定で、雑然とし切っていない。

## 抽象化した設計原則

- 強いビジュアル演出より、情報の優先順位をサイズ差と色の限定運用で作る。
- 画面全体を同一トーンの「面」で揃え、情報ブロックの反復で掲示板らしい秩序を作る。
- スレタイの前後に、読む前の判断材料を置く。中身に入る前の比較効率が高いほど一覧は気持ちよく見える。
- ライブ感は派手なアニメーションではなく、更新情報の専用レーンを持つことで成立する。
- 小さい文字を使う場合も、全部を同じ小ささにせず、主役だけは明確に大きくする。
- 操作導線は常にあるが、本文より強くしない。掲示板の主役はあくまで会話である。
- 少し古い Web らしさは、未整理さではなく「軽い密度感」と「箱の反復」で生まれる。

## BattleBoard の匿名掲示板 UI への具体案

- 全体背景を単色ベタから外し、薄い紙感やノイズ感を持つニュートラル背景にする。カード単体ではなく「板の上に投稿が並ぶ」空気を先に作る。
- スレ一覧に `勢い` `参加人数` `新着` などの小さな情報帯を追加し、タイトルの左か右に固定幅で並べる。数値は小さくても比較しやすくする。
- スレタイだけは一段強くし、ヘッダや補助情報は今より抑える。すでに調整中の投稿ヘッダ縮小は、この方向と整合する。
- トップまたはスレ一覧上部に「いま伸びているレス」や「直近更新」を流すヘッドライン帯を置く。AI混入ゲームの緊張感とも相性がいい。
- 投稿カードはモダンな大きいカード感より、境界が軽く見える箱の連続に寄せる。角丸・影を弱め、面と線で密度を出す方向が合う。
- 強調色を増やしすぎず、危険・注目・ライブ状態だけに限定する。通常時は灰系、重要状態だけ暖色を使うと匿名掲示板らしい落ち着きが出る。
- 「読む」「最新」「返信」などの補助導線は常設しつつ、本文の近くで静かに見せる。UI の存在感を下げ、会話の連なりを主役にする。
