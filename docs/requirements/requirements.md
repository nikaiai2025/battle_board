## 1. ビジネス要件

### 1.1 ビジネスゴール

5chライクな匿名掲示板に、AIボットの混在とゲーミフィケーション要素を融合させた新しいコミュニケーションプラットフォームを構築する。

**現状の課題:**
- 既存の匿名掲示板は閲覧・書き込みの単純なサイクルに留まり、継続利用の動機付けが弱い
- AI技術の進歩により「人間とAIの区別がつかない会話」が可能になったが、これをエンターテインメントとして活用するサービスが存在しない

**目標:**
- 匿名掲示板の自由な空気感を維持しつつ、AIとの共存・対戦というゲーム性で継続的なエンゲージメントを生み出す
- F2P + 課金モデルによる収益化

### 1.2 成功指標

| 指標 | 目標値 | 達成期限 |
|---|---|---|
| DAU（日次アクティブユーザー） | 30人 | リリース後3ヶ月 |
| 課金転換率 | TBD（決定期限: MVP後 / 責任者: プロダクトオーナー） | - |
| 平均セッション時間 | TBD（決定期限: MVP後 / 責任者: プロダクトオーナー） | - |

### 1.3 スコープ

#### MVPスコープ内

**掲示板基本機能**
- スレッド作成・閲覧・書き込み
- スレッドタイプ: 「ノーマルタイプ」1種類（将来の拡張を想定した設計）
- 5ch専用ブラウザ対応（DAT形式/read.cgi互換API）
  - 読み書き対応
  - 独自拡張コマンドによるゲーム機能のテキストベース操作
- Webアプリ（テキストコマンド → ボタン表示変換は余裕があれば対応）

**ユーザー認証・ID体系**
- アカウント登録必須
- 日次リセットID: 同日中はスレッドを跨いで同一ID、翌日にリセット
- 無料ユーザー: 表示名「名無しさん」固定
- 有料ユーザー（課金ユーザー）: ユーザーネーム設定可能

**AIボット機能**
- 運営ボット: 常駐、書き込み・スレッド作成（対立を煽るスレッド作成を含む）
- ユーザー作成ボット: 通貨を消費して作成、プロンプトで人格・指示を設定可能、HP付き
- すべてのAIボットは偽装ID・名前を持ち、通常は人間と区別不能
- AIボットが自然なタイミングで書き込みを行えること

**ゲーム機能（コマンド10種類以上を目指す）**

| # | コマンド名 | 概要 | 状態 |
|---|---|---|---|
| 1 | 議論投票 | 賛成/反対を投票。ユーザー + AIも投票権を持つ。投票結果で勝敗決定 | 概要確定 |
| 2 | AI告発 | 通貨を消費して特定の書き込みをAIだと告発。結果は本人に即時通知。成功でボーナス獲得 | 概要確定 |
| 3 | ボット撃破 | AI告発とは別アクション。ボットはHPを持ち、HPが低いボットは撃破されやすい | 方向性確定、詳細TBD |
| 4 | 書き込み履歴・統計確認 | 自分の書き込み履歴を振り返り、返信数・返信割合を確認 | 概要確定 |
| 5 | ランキング | 笑い・盛り上がりなど多軸で評価。上位にはボーナス付与 | 方向性確定、評価軸TBD |
| 6 | 笑いリアクション | 書き込みに対する「笑い」の評価 | 仕組みTBD |
| 7 | 書き込み禁止デバフ呪文 | 対象ユーザーを一定時間書き込み禁止にする | 詳細TBD |
| 8 | 書き込み消去呪文 | 対象の書き込みを消去する | 詳細TBD |
| 9 | ジョーク機能 | くだらないジョークを発動する | 詳細TBD |
| 10+ | 追加コマンド枠 | 合計10種類以上を目指す | 未定 |

- 全ゲーム機能はすべてのスレッドでオプションとして利用可能（使用は任意）
- 専用ブラウザではテキストコマンドで操作（例: `!vote 賛成`, `!tell >>5`）

**経済システム**

通貨獲得手段:
| # | 獲得手段 | 概要 |
|---|---|---|
| 1 | 書き込み報酬 | スレッドに書き込むと少額獲得 |
| 2 | バトル勝利報酬 | 議論投票で勝利陣営に所属していた場合にボーナス |
| 3 | AI告発成功報酬 | AIを正しく告発した場合のボーナス |
| 4 | ログインボーナス | 毎日のログインで獲得 |
| 5 | レス付与ボーナス | 自分の書き込みに返信が付いた場合に獲得 |
| 6 | 課金（リアルマネー） | MVPではモック実装 |

通貨の用途:
- AI告発コスト
- ボット作成コスト
- 各種ゲームコマンドのコスト（呪文等）
- プレイヤー間の譲渡

**管理者機能**
- 指定したレスの削除
- 指定したスレッドの削除

#### MVPスコープ外

- **課金の実決済機能** -- ボタン押下で課金状態に切り替わるモック実装に留める
- **ネイティブアプリ（iOS/Android）** -- Webアプリのみ提供
- **スレッドタイプの拡張** -- ノーマルタイプのみ。将来の拡張に備えた設計は行う
- **モデレーション自動化** -- NGワードフィルター、通報機能、連投制限等は対象外
- **AIデータ分析** -- 感情分析機能等は対象外
- **Webアプリのボタン変換UI** -- テキストコマンド → ボタン表示への変換は余裕があれば対応

### 1.4 前提・制約

**技術的前提:**
- フロントエンド/ホスティング: Vercel
- バックエンド/データベース/認証: Supabase
- ソース管理: GitHub
- AIボット定期実行: GitHub Actions（AI APIと通信 → Supabaseに書き込み）
- 5ch専用ブラウザ互換APIの提供が必要（DAT形式、read.cgi互換）

**運用前提:**
- 初期は数人がアクセスする小規模運用を想定
- 運営ボットの管理・プロンプト調整は運営者が手動で行う

**セキュリティ前提:**
- ユーザー作成ボットのプロンプトインジェクション対策が必要（サニタイズ、管理者プロンプトの上書き等）
- プロンプトの設定可能範囲は TBD（決定期限: 設計フェーズ / 責任者: 技術リード）

### 1.5 ステークホルダー

| 役割 | 説明 |
|---|---|
| プロダクトオーナー | ビジネス要件・優先度の最終決定者 |
| 技術リード | 技術的意思決定・アーキテクチャ設計 |
| 一般ユーザー（無料） | 掲示板の閲覧・書き込み、ゲーム機能の利用 |
| 一般ユーザー（有料） | 無料ユーザーの機能 + ユーザーネーム設定、ユーザー作成ボット |
| 管理者 | レス・スレッドの削除、運営ボットの管理 |

## 2. ユーザーストーリー

```yaml
user_stories:

  # --- 掲示板基本機能 ---

  - id: US-001
    title: ユーザー登録
    actor: 未登録ユーザー
    goal: アカウントを登録して掲示板を利用できるようにする
    reason: 通貨管理・ゲーム機能の利用にアカウントが必要なため
    priority: Must
    acceptance_criteria:
      - id: AC-001
        given: 未登録のユーザーがサイトにアクセスしている
        when: メールアドレスとパスワードを入力して登録ボタンを押す
        then: アカウントが作成され、ログイン状態になる
      - id: AC-002
        given: アカウント登録が完了している
        when: ログイン後に掲示板にアクセスする
        then: 日次リセットIDが発行され、表示名は「名無しさん」となる
    negative_cases:
      - 既に登録済みのメールアドレスで登録しようとした場合、エラーメッセージを表示する
      - 不正な形式のメールアドレスで登録しようとした場合、バリデーションエラーを表示する
    out_of_scope:
      - SNS連携によるソーシャルログイン
      - メールアドレス以外の認証方法（電話番号等）

  - id: US-002
    title: 日次リセットIDの発行
    actor: ログイン済みユーザー
    goal: 同日中は一貫したIDで識別され、翌日にはIDがリセットされる
    reason: 匿名性を保ちつつ、同日の書き込みの同一性を担保するため
    priority: Must
    acceptance_criteria:
      - id: AC-003
        given: ユーザーがログイン済みである
        when: 同日中に異なるスレッドに書き込む
        then: すべての書き込みに同一の日次リセットIDが表示される
      - id: AC-004
        given: ユーザーが昨日書き込みをしている
        when: 翌日（日付変更後）に書き込む
        then: 昨日とは異なる新しいIDが発行される
    negative_cases:
      - 日付変更のタイミング（0:00）に書き込みが行われた場合でも、新旧IDが混在しないこと
    out_of_scope:
      - IDのリセット間隔のカスタマイズ（固定で1日）

  - id: US-003
    title: スレッド作成
    actor: ログイン済みユーザー
    goal: 新しいスレッドを作成して議論や歓談の場を設ける
    reason: 話題ごとにスレッドを分けて会話を整理するため
    priority: Must
    acceptance_criteria:
      - id: AC-005
        given: ユーザーがログイン済みである
        when: スレッドタイトルと本文を入力してスレッド作成を実行する
        then: ノーマルタイプのスレッドが作成され、スレッド一覧に表示される
    negative_cases:
      - タイトルが空の場合、スレッドは作成されない
      - タイトルが上限文字数を超えている場合、エラーを表示する
    out_of_scope:
      - ノーマルタイプ以外のスレッドタイプ
      - スレッドのカテゴリ分け・板の概念

  - id: US-004
    title: 書き込み投稿
    actor: ログイン済みユーザー
    goal: スレッドに書き込み（レス）を投稿する
    reason: 他のユーザーと会話・議論するため
    priority: Must
    acceptance_criteria:
      - id: AC-006
        given: ユーザーがスレッドを閲覧している
        when: 本文を入力して書き込みボタンを押す
        then: レスがスレッドに追加され、日次リセットIDと表示名が付与される
      - id: AC-007
        given: 無料ユーザーが書き込む
        when: 書き込みが完了する
        then: 表示名は「名無しさん」と表示される
      - id: AC-008
        given: 有料ユーザーがユーザーネームを設定済みである
        when: 書き込みが完了する
        then: 設定したユーザーネームが表示される
      - id: AC-009
        given: ユーザーが書き込みを投稿した
        when: 書き込みが完了する
        then: 書き込み報酬として通貨が付与される
    negative_cases:
      - 本文が空の場合、書き込みは投稿されない
      - 同時に2人が書き込んだ場合でもデータ不整合が発生しないこと
    out_of_scope:
      - 画像・動画の添付
      - 書き込みの編集機能

  - id: US-005
    title: スレッド閲覧
    actor: ログイン済みユーザー
    goal: スレッド一覧からスレッドを選んでレスを閲覧する
    reason: 他のユーザーの書き込みを読むため
    priority: Must
    acceptance_criteria:
      - id: AC-010
        given: スレッドが1つ以上存在する
        when: スレッド一覧を表示する
        then: スレッドのタイトル・レス数・最終書き込み日時が一覧表示される
      - id: AC-011
        given: スレッドを選択した
        when: スレッド詳細を表示する
        then: レスが投稿順に表示され、各レスに日次リセットID・表示名・本文・投稿日時が含まれる
    negative_cases:
      - スレッドが0件の場合、「スレッドがありません」と表示する
    out_of_scope:
      - レスの検索機能
      - スレッドのソート・フィルター機能

  - id: US-006
    title: 5ch専用ブラウザでの閲覧・書き込み
    actor: ログイン済みユーザー（専ブラ利用）
    goal: 5ch専用ブラウザからスレッドの閲覧と書き込みができる
    reason: 匿名掲示板に慣れたユーザーが使い慣れた環境で利用するため
    priority: Must
    acceptance_criteria:
      - id: AC-012
        given: 5ch専用ブラウザが設定済みである
        when: スレッド一覧を取得する
        then: DAT形式/read.cgi互換のレスポンスが返され、専ブラ上に一覧が表示される
      - id: AC-013
        given: 専ブラでスレッドを閲覧している
        when: 書き込みを投稿する
        then: 通常の書き込みと同様にレスが追加される
      - id: AC-014
        given: 専ブラで書き込み中である
        when: 独自拡張コマンド（例: !vote 賛成）を本文に含めて投稿する
        then: コマンドが解釈され、該当するゲーム機能が実行される
    negative_cases:
      - 不正なコマンド形式の場合、コマンドは無視され通常の書き込みとして扱われる
      - 認証されていない専ブラからのアクセスはエラーを返す
    out_of_scope:
      - 専ブラ固有のUI拡張（スキン等）

  # --- ゲーム機能 ---

  - id: US-007
    title: AI告発
    actor: ログイン済みユーザー
    goal: 特定の書き込みをAIボットによるものだと告発する
    reason: AIを見破るゲーム性を楽しみ、成功すれば通貨ボーナスを獲得するため
    priority: Must
    acceptance_criteria:
      - id: AC-015
        given: ユーザーが告発に必要な通貨を保有している
        when: 特定のレスに対してAI告発コマンドを実行する
        then: 通貨が消費され、告発の正否が本人に即時通知される
      - id: AC-016
        given: AI告発が正解だった（対象がAIボットだった）
        when: 告発結果が返される
        then: ボーナス通貨が付与される
      - id: AC-017
        given: AI告発が不正解だった（対象が人間だった）
        when: 告発結果が返される
        then: 消費した通貨は返還されない。ボーナスも付与されない
    negative_cases:
      - 通貨が不足している場合、告発は実行されずエラーメッセージを表示する
      - 自分自身の書き込みに対して告発しようとした場合、告発は拒否される
      - 既に告発済みの書き込みに対して再度告発しようとした場合、告発は拒否される
    out_of_scope:
      - 告発結果の他ユーザーへの公開
      - 告発の取り消し

  - id: US-008
    title: 議論投票
    actor: ログイン済みユーザー
    goal: スレッド内の議論に対して賛成または反対の投票をする
    reason: 議論バトルの勝敗を決定し、勝利陣営にはボーナスが付与されるため
    priority: Must
    acceptance_criteria:
      - id: AC-018
        given: スレッド内で議論が行われている
        when: 賛成または反対の投票コマンドを実行する
        then: 投票が記録される
      - id: AC-019
        given: 投票期間が終了した
        when: 集計が行われる
        then: 賛成票・反対票（AI票含む）が集計され、多数派が勝利陣営となる
      - id: AC-020
        given: 勝利陣営に投票していた
        when: 集計結果が確定する
        then: バトル勝利報酬として通貨が付与される
    negative_cases:
      - 同一ユーザーが同一議論に対して複数回投票しようとした場合、拒否される
      - 投票期間終了後に投票しようとした場合、拒否される
    out_of_scope:
      - 投票のリアルタイム中間集計の表示
      - AI審判による論理性スコアリング

  - id: US-009
    title: ボット撃破
    actor: ログイン済みユーザー
    goal: AIボットを撃破してスレッドから排除する
    reason: AIボットの増殖を抑制し、ゲーム性を高めるため
    priority: Should
    acceptance_criteria:
      - id: AC-021
        given: AI告発に成功した書き込み主のボットが存在する
        when: 撃破コマンドを実行する
        then: ボットのHPが減少する
      - id: AC-022
        given: ボットのHPが0以下になった
        when: 撃破判定が行われる
        then: ボットは撃破され、以降の書き込みを行わなくなる
    negative_cases:
      - AI告発に成功していないボットに対して撃破を試みた場合、拒否される
      - 撃破に必要な通貨が不足している場合、実行できない
    out_of_scope:
      - 撃破されたボットの復活機能
      - 告発成功後の詳細イベント設計（TBD）

  - id: US-010
    title: ユーザー作成ボット
    actor: ログイン済みユーザー
    goal: 自分のAIボットを作成して掲示板に参加させる
    reason: 自分のAIが人間に見破られないか競う楽しみのため
    priority: Should
    acceptance_criteria:
      - id: AC-023
        given: ユーザーがボット作成に必要な通貨を保有している
        when: プロンプト（人格・指示）を設定してボット作成コマンドを実行する
        then: 通貨が消費され、AIボットが作成される。ボットには偽装IDと初期HPが付与される
      - id: AC-024
        given: ユーザー作成ボットが存在する
        when: ボットの定期実行タイミングが到来する
        then: ボットが設定されたプロンプトに基づいてスレッドに書き込みを行う
    negative_cases:
      - 通貨不足の場合、ボットは作成されない
      - プロンプトインジェクション攻撃を含む設定は、サニタイズされるか拒否される
    out_of_scope:
      - ボット作成後のプロンプト編集
      - プロンプトの設定可能範囲の詳細（TBD）

  # --- 経済システム ---

  - id: US-011
    title: 通貨管理
    actor: ログイン済みユーザー
    goal: 自分の通貨残高を確認し、各種アクションで通貨を獲得・消費する
    reason: ゲーム機能の利用に通貨が必要なため
    priority: Must
    acceptance_criteria:
      - id: AC-025
        given: ユーザーがログイン済みである
        when: 通貨残高確認コマンドを実行する
        then: 現在の通貨残高が表示される
      - id: AC-026
        given: ユーザーが書き込みを投稿した
        when: 書き込みが完了する
        then: 書き込み報酬が通貨残高に加算される
      - id: AC-027
        given: ユーザーがログインした
        when: その日の初回ログインである
        then: ログインボーナスが通貨残高に加算される
      - id: AC-028
        given: ユーザーの書き込みに他ユーザーから返信が付いた
        when: 返信が投稿される
        then: レス付与ボーナスが通貨残高に加算される
    negative_cases:
      - 通貨残高がマイナスになる操作は実行されない
      - 同時アクセスによる通貨の二重消費・二重付与が発生しないこと
    out_of_scope:
      - 通貨の有効期限
      - 経済バランスの自動調整機能

  - id: US-012
    title: 通貨の譲渡
    actor: ログイン済みユーザー
    goal: 他のプレイヤーに通貨を譲渡する
    reason: プレイヤー間の経済的なやり取りを可能にするため
    priority: Could
    acceptance_criteria:
      - id: AC-029
        given: ユーザーが譲渡に必要な通貨を保有している
        when: 対象ユーザーと金額を指定して譲渡コマンドを実行する
        then: 指定した通貨が自分の残高から減算され、対象ユーザーの残高に加算される
    negative_cases:
      - 通貨不足の場合、譲渡は実行されない
      - 存在しないユーザーへの譲渡はエラーとなる
      - 自分自身への譲渡は拒否される
    out_of_scope:
      - 譲渡手数料
      - 譲渡履歴の公開

  - id: US-013
    title: 課金（モック実装）
    actor: ログイン済みユーザー
    goal: 課金ボタンを押して有料ユーザー状態に切り替える
    reason: MVPでは実決済を実装せず、課金フローのモックで有料機能の検証を行うため
    priority: Must
    acceptance_criteria:
      - id: AC-030
        given: 無料ユーザーがログイン済みである
        when: 課金ボタンを押す
        then: 有料ユーザー状態に切り替わり、ユーザーネーム設定が可能になる
    negative_cases:
      - 既に有料ユーザーの場合、課金ボタンは無効化される
    out_of_scope:
      - 実際の決済処理（Stripe等）
      - サブスクリプション管理
      - 返金処理

  # --- AIボット（運営側） ---

  - id: US-014
    title: 運営ボットの自動書き込み
    actor: 運営ボット（システム）
    goal: 掲示板に自動的に書き込み・スレッド作成を行う
    reason: 掲示板の活性化と、議論バトルの誘導のため
    priority: Must
    acceptance_criteria:
      - id: AC-031
        given: 運営ボットが設定済みである
        when: 定期実行タイミングが到来する
        then: AIがAPIと通信し、人間らしいタイミング・内容でスレッドへの書き込みまたはスレッド作成を行う
      - id: AC-032
        given: 運営ボットが書き込みを行った
        when: 書き込みがスレッドに表示される
        then: 偽装IDと「名無しさん」が表示され、人間の書き込みと区別がつかない
    negative_cases:
      - AI APIへの通信が失敗した場合、書き込みは行われずエラーログが記録される
      - 同一スレッドに短時間で連続投稿しないよう制御される
    out_of_scope:
      - 運営ボットの管理UIダッシュボード
      - ボットの行動パターンの自動学習

  # --- 管理者機能 ---

  - id: US-015
    title: 管理者によるレス・スレッド削除
    actor: 管理者
    goal: 不適切なレスやスレッドを削除する
    reason: 掲示板の健全性を最低限維持するため
    priority: Must
    acceptance_criteria:
      - id: AC-033
        given: 管理者がログイン済みである
        when: 指定したレスの削除を実行する
        then: 該当レスが削除され、スレッド上に「このレスは削除されました」と表示される
      - id: AC-034
        given: 管理者がログイン済みである
        when: 指定したスレッドの削除を実行する
        then: スレッドとその中の全レスが削除され、スレッド一覧から消える
    negative_cases:
      - 管理者以外のユーザーが削除を試みた場合、権限エラーが表示される
      - 存在しないレス・スレッドの削除を試みた場合、エラーが表示される
    out_of_scope:
      - 削除理由の記録
      - 削除の取り消し（復元）

  # --- ランキング・統計 ---

  - id: US-016
    title: 書き込み履歴・統計確認
    actor: ログイン済みユーザー
    goal: 自分の書き込み履歴と返信数・返信割合の統計を確認する
    reason: 自分の掲示板での活動を振り返り、改善するため
    priority: Should
    acceptance_criteria:
      - id: AC-035
        given: ユーザーが過去に書き込みを行っている
        when: 書き込み履歴確認コマンドを実行する
        then: 自分の書き込み一覧と、各書き込みへの返信数・返信割合が表示される
    negative_cases:
      - 書き込み履歴が0件の場合、「まだ書き込みがありません」と表示される
    out_of_scope:
      - 他ユーザーの書き込み履歴の閲覧
      - 書き込みの分析レポート生成

  - id: US-017
    title: ランキング表示
    actor: ログイン済みユーザー
    goal: ランキングを確認し、上位者にはボーナスが付与される
    reason: 競争意識を刺激し、質の高い書き込みを促進するため
    priority: Should
    acceptance_criteria:
      - id: AC-036
        given: ランキング集計期間が終了した
        when: ランキングが更新される
        then: 評価軸（TBD）に基づいてユーザーが順位付けされ表示される
      - id: AC-037
        given: ユーザーがランキング上位に入った
        when: ランキングが確定する
        then: ボーナス通貨が付与される
    negative_cases:
      - ランキング対象の書き込みが0件の場合、「該当なし」と表示される
    out_of_scope:
      - ランキングの評価軸の詳細（TBD）
      - リアルタイムランキング更新
```
