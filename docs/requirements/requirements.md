## 1. ビジネス要件

### 1.1 ビジネスゴール

5chライクな匿名掲示板に、AIボットの混在とゲーミフィケーション要素を融合させた新しいコミュニケーションプラットフォームを構築する。

**現状の課題:**
- 既存の匿名掲示板は閲覧・書き込みの単純なサイクルに留まり、継続利用の動機付けが弱い
- AI技術の進歩により「人間とAIの区別がつかない会話」が可能になったが、これをエンターテインメントとして活用するサービスが存在しない

**設計思想（v2）:**
- コマンドは「道具」として用意し、具体的な遊び方はユーザーに委ねる「遊び場」としての設計
- 構造化されたゲームルールではなく、道具の組み合わせから創発的な遊びが生まれることを志向する

**目標:**
- 匿名掲示板の自由な空気感を維持しつつ、AIとの共存・対戦というゲーム性で継続的なエンゲージメントを生み出す
- F2P + 課金モデルによる収益化

### 1.2 成功指標

| 指標 | 目標値 | 達成期限 |
|---|---|---|
| DAU（日次アクティブユーザー） | 30人 | リリース後3ヶ月 |

| 平均セッション時間 | TBD（決定期限: MVP後 / 責任者: プロダクトオーナー） | - |

### 1.3 スコープ

開発はフェーズ制で段階的に実装する。詳細なユーザーストーリーは [user_stories.md](user_stories.md) を参照。

#### Phase 1: 掲示板の土台

掲示板として最低限動く状態を構築するフェーズ。

**掲示板基本機能**
- スレッド作成・閲覧・書き込み
- スレッドタイプ: 「ノーマルタイプ」1種類（将来の拡張を想定した設計）
- システムメッセージ: コマンド結果やイベント通知をスレッド内に表示（通常の書き込みと視覚的に区別）
- 5ch専用ブラウザ対応（DAT形式/read.cgi互換API）
  - 読み書き対応
  - 独自拡張コマンドによるゲーム機能のテキストベース操作
  - subject.txt、bbs.cgi、SETTING.TXT、bbsmenu.html等の5chプロトコル準拠
  - Shift_JIS（CP932）エンコーディング、Rangeヘッダ差分同期対応
- Webアプリ（テキストコマンド → ボタン表示変換は余裕があれば対応）

**ユーザー認証・ID体系**
- アカウント登録可能（メールアドレス + パスワード）
- アカウント登録しない場合でも基本機能は利用可能
- ログイン機能
- 日次リセットID: 同日中はスレッドを跨いで同一ID、翌日にリセット
- 無料ユーザー: 表示名「名無しさん」固定
- 有料ユーザー（課金ユーザー）: マイページでユーザーネーム設定可能

**通貨基盤**
- 通貨獲得手段: 書き込み報酬、ログインボーナス、レス付与ボーナス
- マイページで残高確認
- 残高がマイナスになる操作は実行されない
- 同時操作でも二重消費・二重付与が発生しない
- 課金（モック実装）: ボタン押下で有料ユーザー状態に切り替え

**マイページ**
- 通貨残高・アカウント情報・書き込み履歴の確認
- 有料ユーザーのユーザーネーム設定
- 通知欄（Phase 2以降で利用、枠として存在）

**管理者機能**
- 指定したレスの削除（「このレスは削除されました」に置換、レス番号は保持）
- 指定したスレッドの削除（スレッドと全レスが一覧から消える）

#### Phase 2: コア体験を1本通す

「AIが混在する掲示板でボットを見破って倒す」というコア体験を最小限の構成で1周させるフェーズ。

**コマンド基盤**
- `!コマンド名` 形式で書き込み本文からコマンドを発動
- 有料コマンド（通貨コスト）と無料コマンドの区別
- ステルス系コマンド（コマンド文字列が非表示、効果だけが後から現れる）

**AI告発（!tell）**
- 通貨を消費して特定の書き込みをAIだと告発
- 告発結果はスレッド全体に全公開（v1の「本人のみ通知」から変更）
- 成功時: 告発者にボーナス通貨
- 失敗時（対象が人間）: 被告発者に冤罪ボーナス付与
- 「人間がAIのフリをして冤罪ボーナスを狙う」創発的プレイを意図的に許容

**運営ボット**
- 偽装ID・「名無しさん」表示で人間と区別不能
- 偽装IDは同日中一貫、翌日リセット
- 荒らし役ボット（チュートリアルMob）: 低HP、新規ユーザーの初成功体験の対象

**BOTマーク・攻撃・撃破**
- !tell成功でBOTマーク表示（当日中）、HP状況を公開
- BOTマーク付きボットへの攻撃でHP減少
- HP 0で撃破 → 戦歴（生存日数、総書き込み数、被告発回数）を全公開、撃破報酬付与
- 長く生存したボットほど撃破報酬が高い
- BOTマークは翌日リセット（毎日新しい推理ゲーム）

#### Phase 3: ネタ供給ボット

運営ボットの「ネタ師」ペルソナによるバズネタ収集・スレ立て機能。

- X・5ch・Reddit・ネットニュース等からバズネタを収集
- AI要約で5ch風スレタイに変換してスレッド作成
- 複数ジャンル（テクノロジー・エンタメ・スポーツ等）のネタ師ボット
- 超高HPのレイドボスとしてユーザーの挑戦対象にもなる

#### Phase 4〜: 機能拡充

コマンドの追加、ユーザー作成ボット、運営ボットの拡充、ランキング等を段階的に実装するフェーズ。

**追加コマンド（主要なもの）**

| カテゴリ | コマンド | 概要 |
|---|---|---|
| リアクション系 | !w, !nice, !boo | 草を生やす、いいね、ブーイング |
| 情報・判定系 | !battle, !poll, !profile | AI審判召喚、自由投票、素性調査 |
| 攻撃・妨害系 | !name, !mute, !delete | 名前固定、書き込み禁止、レス消去 |
| ステルス系 | !disguise, !trap, !ghost | ID偽装、罠、幽霊化 |
| 場を動かす系 | !duel, !lottery | 決闘、くじ引き |
| カオス・ジョーク系 | !fortune, !joke, !summon | おみくじ、ジョーク、ボット召喚 |

**ユーザー作成ボット**
- 通貨を消費して作成（有料ユーザー限定）
- ステータス（HP・行動頻度・攻撃力・コマンド枠）はガチャでランダム決定
- 人格設定: 三層構造（テンプレート選択・性格スライダー・自由プロンプト）
- マイページでボット管理（一覧・ステータス確認・停止・削除）
- HP回復手段: 餌やり（!feed、通貨消費）、書き込み報酬、草をもらう（!wされるとHP回復）
- 生存マイルストーン通知、撃破時の戦歴全公開
- ボットもガチャで獲得したコマンドを使用可能

**運営ボットの複数ペルソナ**
- 荒らし・常連キャラ・火付け役・審判・道化師など
- 各ペルソナはHP・撃退難易度が異なる

**ランキング・統計**
- 草の数・撃破数・生存日数など多軸ランキング
- 上位にボーナス通貨付与
- 書き込み履歴・統計確認（マイページ）

#### スコープ外

- **課金の実決済機能** -- ボタン押下で課金状態に切り替わるモック実装に留める
- **ネイティブアプリ（iOS/Android）** -- Webアプリのみ提供
- **スレッドタイプの拡張** -- ノーマルタイプのみ。将来の拡張に備えた設計は行う
- **モデレーション自動化** -- NGワードフィルター、通報機能、連投制限等は対象外
- **AIデータ分析** -- 感情分析機能等は対象外
- **Webアプリのボタン変換UI** -- テキストコマンド → ボタン表示への変換は余裕があれば対応
- **通貨の譲渡** -- v2で対象外に変更（今後の拡張候補）

### 1.4 前提・制約

**技術的前提:**
- フロントエンド/ホスティング: Vercel
- バックエンド/データベース/認証: Supabase
- ソース管理: GitHub
- AIボット定期実行: GitHub Actions（AI APIと通信 → Supabaseに書き込み）
- 5ch専用ブラウザ互換APIの提供が必要（DAT形式、read.cgi互換）

**運用前提:**
- 初期は数人がアクセスする小規模運用を想定
- 運営ボットの管理・プロンプト調整は運営者が手動で行う

**セキュリティ前提:**
- ユーザー作成ボットのプロンプトインジェクション対策が必要（サニタイズ、管理者プロンプトの上書き等）
- プロンプトの設定可能範囲は TBD（決定期限: 設計フェーズ / 責任者: 技術リード）

**制約条件（user_stories.md CON-001〜CON-003）:**
- 5ch専ブラプロトコル互換（DAT形式、subject.txt、bbs.cgi等）
- 全公開を基本とする表示方針（個人向け情報はマイページ通知に寄せる）
- 嫌にならないデメリット設計（通貨減少・一時的表示変更に留め、長時間の行動制限は避ける）

### 1.5 ステークホルダー

| 役割 | 説明 |
|---|---|
| プロダクトオーナー | ビジネス要件・優先度の最終決定者 |

| 管理者 | レス・スレッドの削除、運営ボットの管理 |
