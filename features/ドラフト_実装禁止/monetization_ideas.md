# マネタイズアイデア集

> ドラフト。実装にはfeatureシナリオ追加・要件定義反映が必要。

---

## 1. テーマ/スキン販売

掲示板UIの見た目をユーザーが切り替え。Discord Nitroモデル。

- 無料テーマ数種 + 有料テーマ（単品 or サブスク）
- CSS変数ベース。現行アーキテクチャで実現可能
- 将来的にUGCスキン（ユーザー作成テーマの販売）も視野

### 実装フェーズ

| Phase | 内容 | 規模 |
|---|---|---|
| 1 | テーマ切り替え機構（CSS変数 + `users.theme_id`） | 小 |
| 2 | テーマプリセット数種追加 | デザイン作業 |
| 3 | 課金ゲート（Stripe等） | 中 |
| 4 | スレッド別テーマ / UGCスキン | 大 |

先行事例: Discord Nitro, LINE着せかえ, Reddit Premium

## 2. 名前欄の装飾（コテハン課金）

5chの文化と直結。名前欄・トリップの見た目を課金で差別化。

- 名前の色変え / グラデーション
- 特殊トリップ記号（王冠、星等）
- 名前欄の背景色・枠線

掲示板ならではの課金要素。専ブラでも反映可能（DAT形式にHTML装飾を載せる）。

---

## 8. レス装飾（背景色・文字色）

自分のレスの見た目を課金でカスタマイズ。

- レス背景色: ユーザー設定として `users` に保持。全投稿に一律適用
- 文字色全体: 同上
- **投稿単位の部分色変えはスコープ外**（マークアップ導入・パーサー・XSS対策・専ブラDAT互換と影響範囲が大きすぎる）

### 設計上の注意: コマンドシステムとの棲み分け

```
コマンド:  投稿テキストに埋め込む → 投稿ごとに異なる
レス装飾: users テーブルの設定値  → 全投稿に一律適用（プロフィール設定）
```

「自分のレスは常にこの色」というユーザー設定の位置づけにすることで、
コマンドとはレイヤが分離され競合しない。
「この投稿だけ赤くしたい」はコマンドと食い合うのでやらない。

## 3. AI告発の追加チャンス

BattleBoard固有のゲーム要素を課金に直結。

- 無料ユーザー: 1日N回の告発権
- 課金ユーザー: 追加告発権、告発精度ヒント等
- Pay-to-Win に見えないバランス調整が肝

---

## 4. カスタムAIボット

ユーザーが自分のAIボットの性格・口調をカスタマイズ。

- 無料: 定型ボットのみ
- 課金: 口調テンプレート選択、アイコンカスタマイズ
- セキュリティ制約あり（プロンプトサニタイズ必須、CLAUDE.md記載）

---

## 5. 統計・分析ダッシュボード

自分の投稿・告発・撃破の成績を可視化。

- 勝率、告発成功率、被告発回数等
- 無料: 基本統計のみ
- 課金: 詳細分析、ランキング、傾向グラフ

---

## 6. 広告非表示

王道だが確実な収益源。

- 無料ユーザーにはバナー広告表示
- 課金で広告非表示
- 他の課金メニューとセットでサブスクにまとめるのが自然

---

## 7. スレッド作成の優先・装飾

スレ立て機能の拡張。

- スレッドタイトルの装飾（色、太字等）
- スレッド一覧での上位固定（期間限定）
- スレッド作成数の上限緩和

---

## 9. サウンド・読み上げ（Web版）

掲示板にエンタメ性を追加。Web Audio API / Web Speech API で実現可能。

- 効果音: 新着レス通知音、告発ヒット音、ボット撃破音等
- レス読み上げ（TTS）: Web Speech API or 外部TTS API
- 無料: 基本通知音のみ / 課金: 効果音パック、読み上げ機能アンロック
- 対象はWeb UIのみ（専ブラは対象外）

---

## 10. コイン（ゲーム内通貨）の直接販売

現行のインセンティブ通貨を課金と接続する。ソシャゲの王道モデル。

- ゲーム内で稼ぐ（書き込み・告発成功等）ルートは維持
- 課金で追加購入できるルートを開く
- コインの使い道（コマンド・装飾アンロック等）が増えるほど価値が上がる

### コマンドとの関係

コマンドと課金メニューは機能的に重なりうるが、問題ではない。
むしろコインが両者を繋ぐ共通通貨として機能する:

```
無課金ユーザー: ゲーム内活動 → コイン獲得 → コマンド/装飾に使用
課金ユーザー:   リアルマネー → コイン購入 → 同上（時短）
```

P2Wにならないよう「課金でしか手に入らないもの」は見た目系に限定し、
ゲーム有利系（告発権等）はゲーム内で稼げる範囲に留めるのが定石。

## 優先度の考え方

| 優先度 | アイデア | 理由 |
|---|---|---|
| 高 | 1. テーマ/スキン | 技術的に小さく始められる。実績あるモデル |
| 高 | 6. 広告非表示 | 実装が単純。サブスクの基盤になる |
| 高 | 8. レス装飾 | users設定のみで実装軽量。コマンドと棲み分け済み |
| 中 | 2. 名前欄装飾 | 掲示板文化との親和性が高い |
| 中 | 5. 統計ダッシュボード | ゲーム要素の魅力を強化 |
| 低 | 3. AI告発追加 | P2Wバランスの設計が難しい |
| 低 | 4. カスタムボット | セキュリティ設計が重い |
| 低 | 7. スレッド装飾 | スパム化リスクの制御が必要 |
| 中 | 9. サウンド/読み上げ | エンタメ性向上。Web APIで実現可能 |
| 高 | 10. コイン直接販売 | 全課金メニューの基盤。既存通貨システムの延長 |
